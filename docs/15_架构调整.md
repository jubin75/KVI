2️⃣ Triple Retrieval Engine（DRM + Entity Prune）
class TripleRetrievalEngine:

    def __init__(self, triple_store, drm_model):
        self.triple_store = triple_store
        self.drm_model = drm_model

    def score_triple(self, query: str, triple: Triple) -> float:
        return self.drm_model.score(
            query,
            triple.sentence
        )

    def retrieve_top_m(
        self,
        query: str,
        top_m: int = 50
    ) -> list[Triple]:

        triples = self.triple_store.get_all()

        for t in triples:
            t.score = self.score_triple(query, t)

        triples.sort(key=lambda x: x.score, reverse=True)

        return triples[:top_m]

3️⃣ Relation Gating Engine

替代 ToG-2 Relation Prune

class RelationGatingEngine:

    def group_by_relation(
        self,
        triples: list[Triple]
    ) -> dict[str, RelationGroup]:

        groups = {}

        for t in triples:
            if t.relation not in groups:
                groups[t.relation] = RelationGroup(t.relation)

            groups[t.relation].triples.append(t)
            groups[t.relation].score += t.score

        return groups

    def select_top_k_relations(
        self,
        groups: dict[str, RelationGroup],
        top_k: int = 3
    ) -> list[RelationGroup]:

        sorted_groups = sorted(
            groups.values(),
            key=lambda x: x.score,
            reverse=True
        )

        return sorted_groups[:top_k]

五、Sentence Grounding 模块

用于最终生成约束。

class SentenceGrounding:

    def collect_sentences(
        self,
        triples: list[Triple],
        max_sentences: int = 10
    ) -> list[str]:

        sorted_triples = sorted(
            triples,
            key=lambda x: x.score,
            reverse=True
        )

        return [
            t.sentence
            for t in sorted_triples[:max_sentences]
        ]

    def build_prompt(
        self,
        query: str,
        sentences: list[str]
    ) -> str:

        context_block = "\n".join(sentences)

        return f"""
        Use the following knowledge to answer:

        {context_block}

        Question:
        {query}

        Answer:
        """

六、完整主流程（核心控制逻辑）
def answer_query(
    query: str,
    model,
    triple_store,
    drm_model,
    kvi_config: KVILayerConfig
):

    self_proof = SelfProofController()

    if not self_proof.should_retrieve(query):
        return model.generate(query)

    # 1️⃣ Triple Retrieval
    retrieval_engine = TripleRetrievalEngine(
        triple_store,
        drm_model
    )

    top_triples = retrieval_engine.retrieve_top_m(
        query,
        top_m=50
    )

    # 2️⃣ Relation Gating
    gating_engine = RelationGatingEngine()
    groups = gating_engine.group_by_relation(top_triples)
    selected_relations = gating_engine.select_top_k_relations(
        groups,
        top_k=3
    )

    # 3️⃣ KVI Injection
    injector = KVIInjector()
    gating_policy = KVIGatingPolicy(kvi_config)

    for group in selected_relations:
        layers = gating_policy.get_layers_for_relation(
            group.relation
        )
        injector.inject_relation_bias(model, group, layers)

    injector.inject_subject(
        model,
        top_triples,
        gating_policy.get_subject_layers()
    )

    injector.inject_object(
        model,
        top_triples,
        gating_policy.get_object_layers()
    )

    # 4️⃣ Sentence Grounding
    grounding = SentenceGrounding()
    sentences = grounding.collect_sentences(top_triples)

    prompt = grounding.build_prompt(query, sentences)

    # 5️⃣ Final Generation
    return model.generate(prompt)

九、v4 实现记录（2026-01-25）

问题：v3 实验证明 Scheme C (GraphRAG + 三元 KVI) 效果不如纯 RAG。
根因：graph walk 返回的所有 triple 的 KV 被无过滤地注入 attention，
包含大量与查询无关的 triple（如查症状时注入了 located_in、treats 等）。

修复：严格按照本文档第二、三、八节设计原则实现 DRM -> Relation Gating -> KVI：

    Graph Walk -> DRM Score -> Relation Gating -> KV Budget -> KV Assembly

1 DRM Scoring（对应第二节 Triple Retrieval Engine）
- 使用中文 bigram + 英文词重叠度作为轻量 DRM
- 对 graph walk 返回的每个 triple 的 provenance sentence 与 query 计算相关性分数
- 低于 drm_threshold（默认 0.05）的 triple 被过滤

2 Relation Gating（对应第三节 Relation Gating Engine）
- 按 relation type 分组，计算组内 DRM 分数总和
- 只保留 top-k（默认 2）个 relation 组
- 阻止低分 relation 类型的 KV 进入 attention

3 KV Budget（新增预算控制）
- 从选中的 relation 组中，按 DRM 分数取 top-N（默认 3）个 triple
- Subject anchor 始终注入（不受预算限制）
- 对应 CLI 参数：--max_kv_triples, --drm_threshold, --top_k_relations

4 其他修复
- graph_retriever.py: fallback walk 限制为 1 hop（不再无限制 walk）
- triple_kv_compiler.py: walk_triple_ids=None 或 [] 均不再 fallback 到全量注入
- Evidence sentences 也按 DRM score 排序，高分证据排在 prompt 前面
- 前端 UI 展示 DRM 分数和过滤过程
