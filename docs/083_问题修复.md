# PATCH PROMPT：Gate 铁律（最小修复版）

## 背景（问题复现）

当前系统在以下状态下仍然允许 FACTUAL_ASSERTION：

- Pattern 已匹配（如 "X 是什么"）
- semantic_instances 已生成
- **所有 required slots 均为空**
- Gate 输出：
  - allowed_answer_capabilities 包含 FACTUAL_ASSERTION
  - final_answer_style = FACTUAL_ASSERTION

这会导致模型在 **无证据、无语义实例** 的情况下进行事实断言（hallucination）。

---

## 本次 PATCH 的唯一目标（不可扩展）

> **当所有 required slot 均未被实例化时，Gate 必须禁止 FACTUAL_ASSERTION**

不讨论：
- intent
- surface_forms
- Pattern 粒度
- Evidence 检索质量
- Prompt 模板设计

---

## 铁律定义（必须严格实现）

### 规则 R1（核心）

```text
IF
  pattern.required_slots 非空
AND
  对于所有 required_slots：
    semantic_instances 中对应 slot 的实例数量 == 0
THEN
  allowed_answer_capabilities MUST NOT include FACTUAL_ASSERTION

规则 R2（决策降级）

在触发 R1 的情况下：

若 EXPLANATION 在 allowed_answer_capabilities 中：

final_answer_style = EXPLANATION

decision = ALLOW

否则：

decision = REFUSE

decision_reason = "required slots missing; factual assertion forbidden"

实现位置（仅限以下文件）
1️⃣ external_kv_injection/src/pattern_pipeline.py

在 IntrospectionGate 内部决策逻辑中加入：

required_slots 缺失检测

FACTUAL_ASSERTION 剔除逻辑

⚠️ 不得在 PatternMatcher / SemanticInstanceBuilder 中实现
⚠️ 不得修改 PatternSpec 结构

2️⃣ external_kv_injection/src/runtime/kvi2_runtime.py

无需新增逻辑，只需确保：

Gate 输出的 allowed_answer_capabilities 被严格执行

若 FACTUAL_ASSERTION 不在列表中：

不得以任何形式生成事实型回答

伪代码（作为唯一允许的设计参考）
required = pattern.required_slots
slot_state = semantic_instance.slots

if required:
    missing_all = all(len(slot_state.get(s, [])) == 0 for s in required)
    if missing_all:
        allowed_answer_capabilities.discard("FACTUAL_ASSERTION")

        if "EXPLANATION" in allowed_answer_capabilities:
            final_answer_style = "EXPLANATION"
            decision = "ALLOW"
            decision_reason = "required slots missing; downgrade to explanation"
        else:
            decision = "REFUSE"
            decision_reason = "required slots missing; no safe answer capability"
