# Evidence Sidecar Builder · 最小可用 Patch Prompt

## 目标
新增一个 Evidence Sidecar 生成步骤。
只读 `blocks.enriched.jsonl`，生成 `evidence.sidecar.jsonl`。
不修改、不重建、不依赖 KV Bank / embedding / Pattern / Slot / Gate。

---

## 输入
blocks.enriched.jsonl

---

## 输出
evidence.sidecar.jsonl（jsonl）

---

## EvidenceItem Schema（固定，不可扩展）
```json
{
  "evidence_id": "string",
  "type": "abbreviation | definition | slot_fill",
  "subject": "string",
  "predicate": "string",
  "object": "string",
  "source_block_id": "string",
  "confidence": 0.0,
  "provenance": "blocks.enriched"
}
生成规则
R1 Abbreviation

若 block.abbreviation_pairs 存在：{ "abbr": A, "full": B }
生成：
{
  "type": "abbreviation",
  "subject": A,
  "predicate": "stands_for",
  "object": B,
  "source_block_id": block_id,
  "confidence": 0.9,
  "provenance": "blocks.enriched"
}
R2 Definition

若 block_type == "definition" 或 schema_slots.definition_like == true：
{
  "type": "definition",
  "subject": primary_entity,
  "predicate": "is_defined_as",
  "object": normalized_text,
  "source_block_id": block_id,
  "confidence": 0.7,
  "provenance": "blocks.enriched"
}
primary_entity 优先级：
abbreviation.full > schema_slots.primary_entity > entities[0]

R3 Slot Fill

对每个已填 schema_slot：
{
  "type": "slot_fill",
  "subject": primary_entity,
  "predicate": "has_<slot_name>",
  "object": slot_value,
  "source_block_id": block_id,
  "confidence": 0.6,
  "provenance": "blocks.enriched"
}
去重规则

以 (type, subject, predicate, object) 去重。
不跨 block 合并，不推理、不补全。

脚本约定

建议脚本名：
build_evidence_sidecar_from_enriched_blocks.py

输入：blocks.enriched.jsonl
输出：evidence.sidecar.jsonl
纯离线、幂等。

禁止

不重建 KV Bank

不重新 embedding

不修改上游 schema

不引入检索/ANN
