# 🔧 Minimal Patch Prompt · Fix Gate–Slot–Evidence Deadlock

## 目标（只修跑不通的问题）
在**不重构架构、不引入新模块**的前提下，修复当前系统的三处致命工程断点：
1. Gate 输出状态自相矛盾（REFUSE + FACTUAL_ASSERTION）
2. schema slot missing 时提前 REFUSE，导致无法补证据
3. 无法定位“命中但未被 Slot 消费”的证据块

---

## Patch 1 · Gate 铁律一致性（必须）

**规则**
- 只要 `decision == "REFUSE"`：
  - `final_answer_style = None`
  - 禁止任何 generation / RIM / 注入路径继续执行

**实现**
在 `IntrospectionGate.decide()` 或等价位置追加：

```python
if decision == "REFUSE":
    final_answer_style = None
    allowed_answer_capabilities = []

并在 KVI2Runtime 中强制：
if gate.decision == "REFUSE":
    return refuse_response()

Patch 2 · schema slot missing → 允许一次“非生成型补证据 RIM”

规则

若满足：

decision_reason == "schema-level-slot-limits-answer"

且 schema slots 全部或部分 missing

不得直接终止

允许触发一次 RIM（retrieve-only，不生成）

实现
在 Gate 输出中新增标志：
if schema_slot_missing:
    decision = "REFUSE"
    allow_rim_retry = True

在 Runtime 中：
if gate.decision == "REFUSE" and gate.allow_rim_retry:
    run_rim_retrieval_only()
    revalidate_slots()
    return gate_again()
⚠️ 禁止在此路径下进入 generation。

Patch 3 · Evidence 可见性 Debug（定位 Slot 断裂点）

规则
当 slot_status == missing 时，必须输出：

已检索 block 中

含有 schema_slots / abbreviation_pairs / definition_pairs

但未被 Slot 消费的 block id

实现
在 SemanticInstanceBuilder 或 Slot 校验处追加：
debug["unconsumed_evidence_blocks"] = [
    block.id for block in retrieved_blocks
    if block.has_schema_metadata() and not block.consumed
]
