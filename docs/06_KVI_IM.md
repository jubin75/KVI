# KVI + IM 核心知识速记（Key Notes）

> 本文档用于**长期记忆与快速回顾**，总结 KVI（External KV Injection）与 IM（Introspective Module，内省循环）的关键概念、统一理论框架，以及与 KV cache / KV 空间的本质区分。

---

## 一、一句话总纲（先记住这三句）

1. **KVI 决定模型“能用哪些记忆”**（扩展 attention 的信息空间）
2. **IM 决定模型“何时、是否、如何使用这些记忆”**（控制推理过程）
3. **KV cache 是缓存机制，KV 空间是计算空间**（加速 vs 认知）

---

## 二、KVI（External KV Injection）速记

### 2.1 KVI 是什么？

> **KVI 是一种在不训练主模型参数的前提下，将外部知识直接注入 Transformer attention 的 Key/Value 空间的机制。**

### 2.2 KVI 做的不是检索，而是“记忆注入”

* 检索（retrieval）只是前置步骤
* 真正生效发生在：

[
\text{Attention}(Q, [K_{int}; K_{ext}], [V_{int}; V_{ext}])
]

### 2.3 KVI 解决的核心问题

* 长期记忆受限于 context window
* Prompt / RAG 中外部知识可被忽略
* 精准执行与强条件依赖不稳定

### 2.4 KVI 的本质定位

> **Attention-level memory augmentation（注意力级别的记忆增强）**

---

## 三、IM（Introspective Module）速记

### 3.1 IM 是什么？

> **IM 是一个不直接生成 token，而是对推理过程进行监控、评估和控制的内省模块。**

### 3.2 IM 关注的不是“内容”，而是“过程”

IM 典型关注信号：

* 不确定性 / entropy
* 中间结果冲突
* 失败标志（无法回答、循环、幻觉）

### 3.3 IM 能做哪些决策？

* 是否继续推理
* 是否触发外部记忆检索
* 是否加强 / 减弱 KVI
* 是否更换检索策略或层位

### 3.4 IM 的本质定位

> **Cognitive Controller（认知控制器）**

---

## 四、KVI + IM 的统一理论框架

### 4.1 核心统一思想

> **KVI 提供“可用记忆空间”，IM 控制“推理轨迹”。**

### 4.2 推理的状态演化视角

[
S_{t+1} = f_\theta(S_t, M_t, C_t)
]

* (S_t)：模型隐状态 + KV cache
* (M_t)：KVI 注入的外部 KV（显式记忆）
* (C_t)：IM 输出的控制信号

### 4.3 闭环内省–记忆循环

1. 初始推理（无或弱 KVI）
2. IM 监控推理质量
3. 触发外部记忆需求
4. KVI 注入外部 KV
5. 再推理
6. IM 评估并决定是否收敛

> 这是一个**显式可控的闭环认知系统**。

---

## 五、KV cache vs KV 空间（非常重要）

这是最容易混淆、但也是**区分工程加速与认知增强的关键点**。

### 5.1 KV cache 是什么？

> **KV cache 是 Transformer 在自回归推理中，为避免重复计算而缓存的 past_key_values。**

* 来源：模型对历史 token 的计算结果
* 目的：

  * 跳过 prefill
  * 加速 decode

📌 本质：**性能优化机制**

---

### 5.2 KV 空间是什么？

> **KV 空间是 attention 计算中，Key/Value 所构成的向量集合与其张量结构。**

* 决定 attention 可 attend 的信息范围
* 直接影响：

  * softmax(QKᵀ)
  * 推理路径

📌 本质：**认知与计算空间**

---

### 5.3 二者的根本差异对比

| 维度     | KV cache | KV 空间    |
| ------ | -------- | -------- |
| 关注点    | 计算是否重复   | 计算用什么信息  |
| 作用层级   | 工程 / 系统  | 模型 / 认知  |
| 是否改变推理 | ❌        | ✅        |
| 是否新增知识 | ❌        | ✅（KVI）   |
| 生命周期   | 短期（会话级）  | 可长期（知识级） |

一句话总结：

> **KV cache 复用的是“算过的东西”，KV 空间决定的是“能想什么”。**

---

## 六、激活记忆 / KV cache / KVI 的统一定位

| 模块   | 解决什么   | 本质          |
| ---- | ------ | ----------- |
| 激活记忆 | 推理加速   | KV cache 复用 |
| KVI  | 推理能力增强 | 扩展 KV 空间    |
| IM   | 推理可控   | 状态与策略控制     |

---

## 七、你提出的整体系统一句话总结

> **该系统通过 External KV Injection 构建显式、可更新的记忆空间，并借助 Introspective Module 对推理过程进行闭环控制，在不修改主模型参数的前提下，实现了可控、可解释的长期记忆增强推理。**

---

## 八、快速复习用的“自检问题”

* 外部信息是否进入 attention 的 K/V？（是 → KVI）
* 系统是否只是复用 past_key_values？（是 → KV cache）
* 是否存在监控 + 决策 + 再推理的闭环？（是 → IM）
